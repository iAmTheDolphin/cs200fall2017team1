Design

Coupling and Cohesion:

The Provider Interface class manages methods and operations that the provider
can access from their terminal as well as attributes that store whether if
they are validated and what their ID is. Service Record class allows the
creation of objects and store all the data for a service record. Database
controller is a class that stores the users, service codes, and service records.
Database Controller also provides methods to add, remove, and search from datasets.
The Report class and its subclasses encapsulate relevant data into an object.
User and its subclasses encapsulate the data for the users and allow the management
of members and providers. The software classes are formed according to function,
such as storing and managing data, and handling the providers information and
operations. The classes couple to perform operations on each others datasets.
By using this design, we achieve low coupling and high cohesion with out project.
